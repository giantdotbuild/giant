<section class="pillars">
  <div class="pillar">
    <h3>Language-agnostic</h3>
    <p>
      The engine knows nothing about Go, Rust, or Docker. Targets are just
      <code>inputs → command → outputs</code> in plain <code>giant.yaml</code> files.
    </p>
  </div>
  <div class="pillar">
    <h3>Content-addressed cache</h3>
    <p>
      SHA-256 keys over inputs, environment, and dependency hashes. Local
      cache by default; bring your own bazel-remote, sccache, or S3
      backend for shared builds.
    </p>
  </div>
  <div class="pillar">
    <h3>A core you control</h3>
    <p>
      One static binary that builds, caches, and speaks NDJSON. No JVM, no
      daemon, no plugins in your process. Everything else - tasks, the TUI,
      generation - is a porcelain on your PATH. <a href="/extending/architecture/">Uninstall one and the feature is gone.</a>
    </p>
  </div>
</section>

<section class="belief">
  <Image src={giantGarden} alt="The Giant kneeling in a forest clearing, tending small robot saplings." />
  <div>
    <h2>Built to serve, programmed to care.</h2>
    <p>
      Giant is a quiet build tool. It doesn't run a daemon. It doesn't
      take over your terminal. It doesn't ship a TUI you didn't ask for.
      You run <code>giant build</code> and it finishes. You run it again
      and the cache holds.
    </p>
    <p>
      The whole engine is small enough to read in an afternoon -
      no plugin DLLs, no embedded scripting, no language-specific
      special cases. Every behaviour comes from one of two places:
      your <code>giant.yaml</code> or a porcelain binary on your PATH.
    </p>
  </div>
</section>

<section class="feature-grid">
  <div class="feature">
    <h4>Affected detection</h4>
    <p>Builds only what changed since <code>main</code>, via git diff or an explicit file list.</p>
  </div>
  <div class="feature">
    <h4>Early cutoff</h4>
    <p>Upstream rebuild with byte-identical outputs leaves downstream cache-warm.</p>
  </div>
  <div class="feature">
    <h4>Watch mode</h4>
    <p>Continuous rebuilds with a debouncer that honors your selection patterns.</p>
  </div>
  <div class="feature">
    <h4>Path labels + exclusion</h4>
    <p><code>giant build '//src/...' '!//src/legacy/...'</code> - same language across build, test, watch, affected.</p>
  </div>
  <div class="feature">
    <h4>Tags</h4>
    <p>Mark targets with <code>release</code>, <code>flaky</code>, anything. Filter via <code>--tag</code> / <code>--no-tag</code>.</p>
  </div>
  <div class="feature">
    <h4>An engine you can talk to</h4>
    <p>Builds run over <a href="/guides/controlling-giant/">NDJSON</a>. A TUI, an IDE, or a web app drives Giant without linking its code.</p>
  </div>
  <div class="feature">
    <h4>LRU cache eviction</h4>
    <p>Cache stays under a configured size limit. No <code>giant gc</code> needed; it just works.</p>
  </div>
</section>
